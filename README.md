# spawndir

A tiny command-line tool that spawns a uniquely named workspace directory and prints its path. Directory names are built from a random combination of an adjective, a color, a noun, and a suffix - for example `CalmEmeraldFoxVenture` - so they are memorable yet effectively unique.

Randomness comes from `crypto/rand`, and the word lists are embedded into the binary at build time, so `spawndir` is a single self-contained executable with no runtime dependencies.

> [!WARNING]
> **AI notice:** All code in this repository was fully generated by **GLM-5.2** and **DeepSeek V4 Flash**. Review carefully before use.

## Install

```sh
go install github.com/dartt0n/spawndir
```

Or build from source:

```sh
go build -o spawndir
```

## Usage

```sh
spawndir            # create ~/Workspace/<name> and print the path
spawndir -r ./work  # create ./work/<name> and print the path

cd $(spawndir)      # change the current directory to the new generated directory
```

### Flags

| Flag | Default | Description |
| --- | --- | --- |
| `-r` | `$HOME/Workspace` | Root directory in which to create the new workspace. |

The generated directory is created with `mkdir -p` semantics, so nested roots are created as needed. The full path of the new directory is printed to stdout, making the tool easy to compose into shell pipelines:

```sh
cd "$(spawndir)"
```

## How names are generated

A name is the concatenation of four words, one drawn uniformly at random from each embedded list, with each word guaranteed distinct within the name:

```
<adjective><color><noun><suffix>
```

| List | Count | Example words |
| --- | --- | --- |
| adjectives | 546 | `Abandoned`, `Acoustic`, `Brave` |
| colors | 95 | `Amber`, `Emerald`, `Indigo` |
| nouns | 1920 | `Fox`, `Harbor`, `Vertex` |
| suffixes | 67 | `Venture`, `Atlas`, `Workbook` |

The pool size is roughly `546 × 95 × 1920 × 67 ≈ 6.7 × 10⁹` combinations, so collisions are vanishingly unlikely for any realistic number of workspaces.

## Word sources

The embedded word lists are derived from public-domain sources. See [`SOURCES.md`](SOURCES.md) for full attribution and the derivation process.

> [!NOTE]
> The embedded `.txt` word lists are stored as ordinary Git objects, **not** Git LFS. Do not move them to LFS: the Go module proxy does not run `git lfs pull`, so `go install` would embed the LFS pointer text instead of the actual word lists.
